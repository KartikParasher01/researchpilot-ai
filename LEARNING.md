# Day 1

### What is Git?
#### It is a Version Control System (VCS) that runs locally on your computer and tracks every single change you make to your project's files.

### What is GitHub?
#### GitHub is the website where you upload and store those Git repositories online so you can share them or collaborate with others.    

### What is a virtual environment?
#### When you install packages in Python normally, they get installed "globally" across your entire computer. A virtual environment lets you create a private, self-contained folder that has its own copy of Python and its own isolated set of libraries.

### Why do we use .env?
#### A .env file (short for "environment" file) is a simple text file used in software development to store configuration settings and sensitive secrets (like API keys, database passwords, and tokens) separate from your actual source code.

### Why do we use requirements.txt?
#### In Python projects, a requirements.txt file is a simple text file that lists all the external libraries (dependencies) your project needs to run, along with their specific versions.

### Why do we have src/?
####  Having a dedicated src/ (source) directory is an industry-standard best practice in software engineering. While Python doesn't force you to use it—you could technically dump all your files directly into the root folder—separating your code into a src/ directory solves several critical problems as your project grows.

### What is the purpose of README.md? 
#### It tells you about the project, what's been used the project summary, the flow.
